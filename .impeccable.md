## Design Context

### Users
Australian homebuyers, refinancers, and mortgage brokers comparing mortgage rates across 65+ banks. Users are financially motivated, data-literate, and want to find the best rate for their specific situation (variable vs fixed, owner-occupied vs investment, P&I vs IO). They think in terms of banks first ("what is ANZ offering?") then drill into specific products. Time-poor, want answers fast.

### Brand Personality
**Authoritative, transparent, data-forward.** This is a free open-source tool — no ads, no affiliate links, no bias. Trust through density and clarity. Three words: **clear, credible, comprehensive.**

### Aesthetic Direction
**Editorial financial data.** Think Bloomberg Terminal meets Stripe — dense, purposeful typography, clean grids, semantic color use. NOT AI slop (no indigo/violet gradients, no glassmorphism, no generic card grids, no rounded-2xl everything). NOT corporate bank (no stuffy conservatism). Light mode primary, dark mode supported. Single strong accent color with semantic colors for rate trends (green for lower/better, red for higher). Distinctive display typeface for rates. Asymmetric layouts, left-aligned text, varied spacing rhythm.

### Anti-References
- No gradient headers or gradient text
- No identical card grids with icon + heading + description
- No font-mono for "data vibes"
- No border-t-4 colored accent borders
- No hover:-translate-y-0.5 shadow transitions
- No glassmorphism or glowing dark mode
- No generic hero metric layouts

### Design Principles
1. **Data first, decoration never** — Every pixel should convey information. If it doesn't help users compare rates, remove it.
2. **Banks before rates** — Users think in lenders, not rate rows. Organize by bank, let them drill down.
3. **Progressive disclosure** — Show the signal first (best rates, key comparisons), reveal complexity on demand (full product details, historical trends, fine print).
4. **Editorial density** — Don't fear dense information. Financial professionals read dense tables. Make them scannable with typography, spacing, and color — not sparse.
5. **Earned trust through transparency** — Show where data comes from, when it was updated, what's missing. No hidden agendas.
